
- Формула вычисления финальной цены на скин:
FinalPrice = BasePriceUsd + (BTC_USD / 10000)
BTC_USD получаем через сторонний API (через coingecko.com)

- Endpoints
GET    /api/skins - получение всех скинов
GET    /api/skins/{id} - получение конкретного скина по значению id
POST   /api/purchases/{skinId} - покупка скина по значению id
GET    /api/purchases - список купленных скинов

- Для запуска backend просто запустить проект (ссылка в браузере https://localhost:7164/swagger/index.html)

- Для запуска frontend после запуска проекта поменять ссылку на https://localhost:7164/index.html

- Текущая архитектура проекта состоит из классов контроллеров, сервисов и данных. Контроллеры содержат
точки входа для HTTP-запросов, не содержат бизнес-логики и вызыввают сервисы. Сервиса отвечают за бизнес-логику
PriceService рассчитывает финальную цену, а BtcRateService получает курс BTC обращаясь к стороннему API. Класс
с данными InMemoryDatabase по сути просто содержит список скинов и покупок в памяти. Также содержится простой
frontend c отображением списка скинов, показывает финальную цену и позволяет купить скин.

async/await в проекте используется для HTTP-запросов к стороннему API, контроллеров и сервисов.
Контроллеры возвращают Task<IActionResult>, сервисы возвращают Task<T>.

В сервисе BtcRateService добавлено in-memory кэширование на 300 секунд для уменьшения нагрузки на внешний API.
При ошибке используется последнее успешное значение курса.

В проекте используется DI, в частности Constructor Injection - контроллеры и сервисы получают 
зависимости через конструктор. Зависимости регистрируются в Program.cs. Контейнер сам создаёт граф зависимостей.
Используются разные lifetime: Singleton для InMemoryDatabase и Scoped для сервисов, BtcRateService через AddHttpClient.
